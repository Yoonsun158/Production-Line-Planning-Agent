# 目标
设计一个AI agent，用于为不同类型的废旧电池生成回收计划报告，并可视化搭建的产线。


# 描述 

具体而言，
1. 用户给出一种电池，
2. agent根据电池的技术文件(存在在知识库里)，结合电池本身形状、品牌、材料等因素，
3. 设计产线（预设一些可选的工序，每个工序为一个工艺单元，用于之后的可视化），
4. 最后生成简要的SOP废旧电池回收报告，以及可视化产线（展示连接哪些工艺单元）

# 需求

## 前端
一个前端的交互界面，包含三个部分。整体大小不要超过一个窗口。可以中英文语言切换
1. 第一部分用于输出简短的报告
    - 是一个文本框
    - 位于交互界面左侧
    - 结构化输出回收报告
        '''
        1.电池名
        <简要分析电池的特性>
        2.产线搭建
        <包含哪些预设的工艺单元，以及连接路径>
        3.回收流程
        <简要描述回收流程和注意事项>

        '''
2. 第二部分实时可视化产线的连接路径
    - 位于交互界面的右侧
    - 初始状态：将知识库中预设的每个工艺单元，可视化为一个合适的标识(例如小盒子，并显示工艺单元的名称)，然后随机产生一个布局。
    - 根据最后输出的报告中'2.产线搭建',可视化连接路径和方向

3. 第三部分包含用户输入框和对话框
    - 整体位于交互界面的中间
    - 用户输入框位于区域的上方，包含一个'发送'按钮
    - 对话框位于区域下方
    - 后端执行过程中的反馈在对话框中打印。

## 后端

搭建用于废旧电池回收的Agent。

### Agent的大致流程
1. 用户将输入框中的信息，发送给LLM
2. LLM根据系统提示词，结合知识库中的向量数据库，按照ReAct (Reason + Act)的逻辑，调用合适的Tool执行action
3. tool执行完后返回 'action-observation' 到'执行历史记录'文件中
4. 将历史记录发给LLM，然后继续执行步骤2和步骤3，直到成功生成回收报告以及可视化产线路径。

注意：执行过程要在对话框中给予用户反馈
